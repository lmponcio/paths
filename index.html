Relative Paths

   by Martin Poncio

  • Below you can find a directory tree representing the files of this static site (see repository)
  • The file containing the HTML you are seeing rendered in the browser is signalled as "
    YOU ARE HERE
    "
  • Hover over each file link to see the relative path that will take you there
  • Click on each file link to go from one page to another and explore the different relative paths
  • Rules to remember:
    • ./ means "current directory" (the directory containing your current file)
    • ../ means "directory containing the current directory", and it can be used more than once: ../../ means "the directory containing the directory that contains the current one"
  • Example: if you are in
    file2.html
    , then
    • ./ is
      dir2
    • ../ is
      dir1
    • ../../ is
      dir0
dir0
├── dir1
│   ├── dir2
│   │   └── file2.html
│   └── file1.html

│   ├── dir4
│   │   ├── dir5
│   │   │   └── file5.html
│   │   └── file4.html
│   └── file3.html
